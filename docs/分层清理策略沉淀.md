# 分层清理策略沉淀

## 背景

项目同时存在四类文件：远端 SFTP 中转音频、本地上传原视频、本地中间缓存、最终成果。它们价值不同，不能统一按一周删除。

目标是减少云服务器和本地磁盘占用，同时不破坏断点续跑、同源缓存复用和最终成果留存。

## 分类与默认保留

| 类型 | 路径 | 默认保留 | 策略 |
|---|---|---:|---|
| 远端中转音频 | `/var/www/class-up/audio/` | 6 小时 | 识别完成后优先主动删除，后台清理兜底 |
| 成功任务本地上传原视频 | `outputs/uploads/**/video.*` | 24 小时 | 任务成功后短期保留排查 |
| 失败任务本地上传原视频 | `outputs/uploads/**/video.*` | 168 小时 | 失败任务保留更久，便于复盘 |
| 本地中间缓存 | 任务目录 `intermediate/` | 168 小时 | 支持断点续跑和同源缓存复用 |
| 最终成果 | `manifest.json`、字幕、文稿、知识点、学习包 | 长期 | 默认不自动删除 |

## 触发方式

- 任务结束轻量清理：Doubao 识别完成后删除本任务远端中转音频。
- 后台定期清理：Web 服务启动后按 `cleanup.background_interval_hours` 扫描。
- 手动清理：CLI 和 Web API 支持预览与执行。

```powershell
python -m class_up.cli cleanup --dry-run --target all
python -m class_up.cli cleanup --execute --target all --reason manual
```

Web API：

- `GET /cleanup/preview`
- `POST /cleanup/run`

## 磁盘压力保护

当本地可用空间低于 `cleanup.disk_min_free_gb` 时，可以提前清理最旧、最低价值缓存。

清理优先级固定：

1. 本地上传原视频。
2. 最旧任务的 `intermediate/audio/` 和 `intermediate/segments/`。
3. 其他中间缓存，例如 `intermediate/transcription/raw/`。

最终成果默认不因磁盘压力自动删除。

## 审计要求

所有清理动作写入：

```text
outputs/system/cleanup_events.jsonl
```

每条事件至少包含：

- `timestamp`
- `trigger`
- `target_type`
- `path`
- `reason`
- `bytes`
- `success`
- `error`

清理失败不能中断主任务，只能作为 warning、manifest review 或审计事件记录。

## 缓存复用兼容

同源缓存复用只匹配仍存在且完整的 `intermediate/`：

- 源视频 `sha256` 一致。
- 切分配置一致。
- 标准化音频和分段文件存在。
- 分段转录 JSON 完整。

如果缓存已被清理，系统自动回退到重新切分和重新转录。
