# 网课助手开发计划

本计划以 [开发目标.md](./开发目标.md) 为依据，分三个里程碑交付，不追求一步到位。

所有阶段的数据结构、状态、路径和交接契约以 [数据协议.md](./数据协议.md) 为准；阶段计划不得另行定义与数据协议冲突的结构。

---

## 里程碑总览

| 里程碑 | 交付内容 | 验收重点 |
|---|---|---|
| M1 | 工程基础 + 转录流水线 | 视频 → SRT + 文稿，端到端跑通 |
| M2 | 知识点结构化 | 文稿 → 知识库 Markdown + JSON |
| M3 | 视频切片 + 学习包 | 按知识点生成切片目录 |

---

## M1：转录流水线

**目标**：给定一个视频文件，输出 `full_subtitle.srt` 和 `full_transcript.txt`。

### 任务

**工程基础**
- [x] 初始化包结构（见开发目标 §8 模块边界）
- [x] CLI 入口：接收视频路径、输出目录、配置文件
- [x] CLI M1 音频准备命令：支持一键视频转音频，输出标准化 `wav` 文件
- [x] 配置加载：`.env` + YAML，含模型服务类型、服务地址、鉴权方式、API Key、上传限制、并发数、超时时间、重试策略、分段时长、音频输出格式
- [x] 初始化标准输出目录，包含中间产物目录、最终产物目录和 manifest 路径
- [x] manifest 读写：记录阶段状态、中间产物路径、错误信息
- [x] ffmpeg 可用性检测

**转录流水线**
- [x] `media/audio.py`：提取音频，标准化为 16kHz mono，音频格式按模型服务配置选择
- [x] `media/audio.py`：提供视频转音频函数，供 M1 音频准备命令和 M1 流水线复用
- [x] `media/audio.py`：按配置时长切段（默认 10 分钟，30 秒重叠）
- [x] `transcription/base.py`：定义转录适配接口
- [x] `transcription/service.py`：接入转录服务适配器，输出符合数据协议的分段转录 `items[]` 结构
- [x] `transcription/doubao.py`：按 [M1 豆包录音识别接入协议](./M1开发阶段/豆包录音识别接入协议.md) 接入新版控制台录音文件识别模型 2.0
- [x] `upload/sftp.py`：上传分段音频到云服务器并生成公网可访问 URL
- [x] `transcription/merge.py`：按 manifest 分段 `index` 合并分段结果，修正时间戳偏移，去除重叠区重复
- [x] `transcription/srt.py`：生成 SRT 文件
- [x] 输出 `full_transcript.txt`
- [x] manifest 记录每个分段状态，支持失败后跳过已完成段重跑
- [x] Web UI API 选择栏支持 `mock` 和 `豆包语音识别`，选择豆包后填写 API Key 即触发豆包 provider

**测试**
- [x] 单元测试：时间码转换、SRT 生成、分段合并逻辑
- [x] 单元测试：CLI `audio` 命令参数解析和旧式 M1 调用兼容
- [x] 单元测试：豆包 `utterances[]` 到内部 `items[]` 映射，Web 选择豆包后的配置默认值和密钥摘要不泄露
- [x] 集成测试：用 mock 转录服务跑通完整流程
- [x] 手工验收：`audio` 命令可将样例视频转换为 16kHz 单声道 `wav`
- [ ] 手工验收：30 分钟样例视频，字幕时间码单调递增，无明显断句错误
- [ ] 手工验收：Web UI 选择豆包并填写 API Key 后跑通真实 M1 端到端

### 验收标准

- 30 分钟以内视频可完整转录，SRT 时间码无乱序。
- M1 音频准备命令可单独完成视频到音频转换，不触发转录。
- 失败时 manifest 记录失败阶段，重跑不重复已完成分段。
- API Key 不出现在任何日志或 manifest 中。

## M2：知识点结构化

**目标**：基于 M1 输出，生成 `knowledge_points.json` 和 `knowledge_base.md`。

依赖：M1 验收通过。

### 任务

- [ ] `analysis/base.py`：定义文本分析适配接口
- [ ] `analysis/service.py`：接入文本分析模型（国产大模型优先）
- [ ] Prompt 设计：输入文稿 + 时间轴，输出知识点 JSON（含 title/start/end/keywords/summary/content）
- [ ] `analysis/knowledge_points.py`：将模型输出规范化为数据协议定义的 `knowledge_points.json`，并做时间范围合法性检查
- [ ] 输出 `knowledge_points.json`
- [ ] 输出 `knowledge_base.md`（每个知识点一个 Markdown 块）
- [ ] manifest 记录本阶段状态

**测试**
- [ ] 单元测试：知识点 JSON 校验、时间范围越界检测
- [ ] 集成测试：mock 文本分析服务跑通完整流程
- [ ] 手工验收：知识点时间范围覆盖合理，Markdown 可直接阅读

### 验收标准

- 每个知识点有合法 start/end，不超出视频总时长。
- JSON 可被 M3 稳定读取（schema 不变）。
- 模型输出格式异常时，记录错误并提示人工修正，不崩溃退出。

## M3：视频切片 + 学习包

**目标**：按知识点生成切片视频，整理为最终课程学习包目录。

依赖：M2 验收通过。

### 任务

**视频切片**
- [ ] `media/video.py`：按 knowledge_points.json 时间范围切割视频
- [ ] `media/video.py`：生成对应 SRT 字幕片段，软挂到切片
- [ ] `packaging/naming.py`：文件名清理，格式 `{index}_{title}_{HH-MM-SS}`

**学习包组织**
- [ ] `packaging/organizer.py`：按输出目录规范（见开发目标 §10）汇总所有产物
- [ ] 每个知识点目录包含 `clip.mp4`、`subtitle.srt`、`notes.md`
- [ ] 生成顶层目录索引
- [ ] manifest 最终状态写入 `success`，并按数据协议的切片记录结构写入所有切片产物路径

**测试**
- [ ] 单元测试：文件名清理函数
- [ ] 集成测试：端到端从视频到完整学习包
- [ ] 手工验收：切片时长与知识点时间范围一致，文件名无非法字符

### 验收标准

- 输出目录结构与开发目标 §10 一致。
- 所有最终产物路径写入 manifest，其中切片产物路径必须写入 manifest 的切片记录结构，不只写 `outputs.clips_root`。
- 文件名不含非法字符，目录可直接归档。

## 风险控制

| 风险 | 应对 |
|---|---|
| 转录时间戳不稳定 | 合并阶段统一校验，中间产物可人工修正后重跑后续阶段 |
| 文本分析输出格式不稳定 | 强制 JSON schema 校验，失败时写入错误文件，不阻断流程 |
| API 限流或中断 | 并发数、重试、超时全部配置化，manifest 支持断点恢复 |
| 豆包标准版只接受公网音频 URL | M1 通过 SFTP 上传分段音频到云服务器，再把公网 URL 传给豆包 |
| ffmpeg 跨平台差异 | M1-M3 只支持 Windows 本机，命令封装集中在 `media/ffmpeg.py` |
| 文件名含非法字符 | 统一经 `packaging/naming.py` 清理，不在业务代码中处理 |
