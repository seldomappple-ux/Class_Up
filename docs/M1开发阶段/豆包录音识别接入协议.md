# M1 豆包录音识别接入协议

## 1. 接入目标

本文定义 M1 阶段接入豆包录音文件识别模型 2.0 的工程协议。

本接入只使用新版控制台 API Key 方式，不使用旧版 `APP ID`、`Access Token`、`Cluster` 或 `X-Api-App-Key` / `X-Api-Access-Key`。

豆包识别结果必须先转换为 `docs/数据协议.md` 第 7 节定义的分段转录结果结构，再进入合并、SRT 和全文文稿生成阶段。

## 2. 接入范围

本协议属于 M1 的转录服务适配范围。

实现目标：

- 将本地音频分段上传到云服务器。
- 通过公网 URL 调用豆包录音文件识别标准版 API。
- 轮询查询识别结果。
- 将豆包返回结果转换为项目内部 `items[]` 结构。
- 保存每段转录 JSON 和必要的原始响应。
- 支持失败记录和断点重跑。

本协议不处理：

- 豆包旧版控制台鉴权。
- 实时语音识别。
- 浏览器直传。
- 自动申请或管理 API Key。
- 云服务器 Nginx 配置和 cron 配置。

## 2.1 当前实现状态

截至 2026-05-19，项目已实现 `doubao` 转录 provider 的后端路径：

- Web UI 的 API 选择栏包含 `豆包语音识别`。
- 用户在 Web UI 选择豆包后，只需要在页面填写豆包新版控制台 API Key。
- 后端将页面填写的 API Key 放入当前进程环境变量 `CLASS_UP_DOUBAO_API_KEY`，不写入 manifest、日志或原始响应文件。
- 后端自动使用豆包默认配置：`endpoint=https://openspeech.bytedance.com`、`model=bigmodel`、`resource_id=volc.seedasr.auc`。
- 后端自动启用 `upload.provider=sftp`，通过 SFTP 上传分段音频后把公网 URL 提交给豆包。
- mock provider 仍保留，用于无密钥本地冒烟测试。

Web UI 不采集 SFTP 服务器信息。SFTP 所需信息必须通过 `.env` 或系统环境变量提供。

## 4. 配置协议扩展

### 4.1 transcription

豆包配置示例：

```yaml
transcription:
  provider: doubao
  endpoint: https://openspeech.bytedance.com
  model: bigmodel
  auth_type: api_key
  api_key_env: CLASS_UP_DOUBAO_API_KEY
  resource_id: volc.seedasr.auc
  upload_limit_mb: 25
  concurrency: 2
  timeout_seconds: 120
  max_retries: 3
  poll_interval_seconds: 1
  max_poll_seconds: 600
  request:
    enable_itn: true
    enable_punc: true
    enable_ddc: false
    enable_speaker_info: false
    enable_channel_split: false
    show_utterances: true
    vad_segment: true
    sensitive_words_filter: ""
```

字段说明：

| 字段 | 必填 | 说明 |
|---|---|---|
| `provider` | 是 | 豆包接入固定为 `doubao` |
| `endpoint` | 是 | 固定为 `https://openspeech.bytedance.com` |
| `model` | 是 | 固定为 `bigmodel` |
| `api_key_env` | 是 | 固定使用 `CLASS_UP_DOUBAO_API_KEY` |
| `resource_id` | 是 | 豆包录音文件识别模型 2.0 固定为 `volc.seedasr.auc` |
| `poll_interval_seconds` | 是 | query 轮询间隔 |
| `max_poll_seconds` | 是 | 单段最长轮询等待时间 |
| `request.show_utterances` | 是 | 必须为 `true`，否则无法稳定生成 SRT 时间轴 |

### 4.2 upload

SFTP 上传配置示例：

```yaml
upload:
  provider: sftp
  host_env: CLASS_UP_UPLOAD_HOST
  port: 22
  username_env: CLASS_UP_UPLOAD_USER
  private_key_path_env: CLASS_UP_UPLOAD_KEY_PATH
  remote_dir: /var/www/class-up/audio/
  public_url_base: https://boneorbit.com/class-up/audio/
```

对应 `.env` 示例：

```dotenv
CLASS_UP_DOUBAO_API_KEY=
CLASS_UP_UPLOAD_HOST=
CLASS_UP_UPLOAD_USER=
CLASS_UP_UPLOAD_KEY_PATH=
```

Web UI 使用约定：

- `CLASS_UP_DOUBAO_API_KEY` 可以由 Web 表单临时写入当前后端进程环境变量。
- `CLASS_UP_UPLOAD_HOST`、`CLASS_UP_UPLOAD_USER`、`CLASS_UP_UPLOAD_KEY_PATH` 必须由 `.env` 或系统环境变量提前配置。
- `CLASS_UP_UPLOAD_KEY_PATH` 指向本机 SSH 私钥文件路径；私钥内容不得写入配置文件、manifest 或文档。

密钥规则：

- `.env` 不提交。
- manifest 不保存 API Key、SSH 私钥内容、Token 或密码。
- manifest 可以保存 `api_key_env`、`host_env`、`username_env`、`private_key_path_env` 这些环境变量名。

## 5. 豆包 API 协议

### 5.1 submit

接口：

```text
POST https://openspeech.bytedance.com/api/v3/auc/bigmodel/submit
```

请求头：

```http
Content-Type: application/json
X-Api-Key: ${CLASS_UP_DOUBAO_API_KEY}
X-Api-Resource-Id: volc.seedasr.auc
X-Api-Request-Id: {uuid}
X-Api-Sequence: -1
```

请求体：

```json
{
  "user": {
    "uid": "class-up"
  },
  "audio": {
    "url": "https://boneorbit.com/class-up/audio/segment-0001.wav",
    "format": "wav",
    "codec": "raw",
    "rate": 16000,
    "bits": 16,
    "channel": 1
  },
  "request": {
    "model_name": "bigmodel",
    "enable_itn": true,
    "enable_punc": true,
    "enable_ddc": false,
    "enable_speaker_info": false,
    "enable_channel_split": false,
    "show_utterances": true,
    "vad_segment": true,
    "sensitive_words_filter": ""
  }
}
```

成功判断：

- `X-Api-Status-Code == 20000000` 表示提交成功。
- submit response body 可以为空，不作为成功判断依据。
- 应保存 `X-Tt-Logid` 用于排查。

### 5.2 query

接口：

```text
POST https://openspeech.bytedance.com/api/v3/auc/bigmodel/query
```

请求头：

```http
Content-Type: application/json
X-Api-Key: ${CLASS_UP_DOUBAO_API_KEY}
X-Api-Resource-Id: volc.seedasr.auc
X-Api-Request-Id: {submit 使用的 uuid}
```

请求体：

```json
{}
```

状态判断：

| `X-Api-Status-Code` | 含义 | 处理 |
|---|---|---|
| `20000000` | 成功 | 解析 body 并转换为内部转录结果 |
| `20000001` | 正在处理中 | 等待后继续轮询 |
| `20000002` | 任务在队列中 | 等待后继续轮询 |
| `20000003` | 静音音频 | 转换为可记录失败，默认不重试 |
| `45000001` | 请求参数无效 | 不重试 |
| `45000002` | 空音频 | 不重试 |
| `45000131` | 提交速度超过限制 | 可重试或降低并发 |
| `45000132` | 音频超过大小限制 | 不重试，需缩短分段 |
| `45000151` | 音频格式不正确 | 不重试 |
| `55000031` | 服务器繁忙 | 可重试 |
| `550xxxx` | 服务内部错误 | 可重试 |

轮询规则：

- 间隔使用 `transcription.poll_interval_seconds`。
- 最长等待使用 `transcription.max_poll_seconds`。
- 超过最长等待后写入 `TRANSCRIPTION_TIMEOUT`。

## 6. 音频上传协议

### 6.1 远端文件命名

上传文件名必须稳定且避免冲突：

```text
{task_id}_{segment_id}.wav
```

示例：

```text
20260519-120000-course_segment-0001.wav
```

### 6.2 上传流程

每个待转录分段：

1. 从 manifest 读取 `segment.audio_path`。
2. 通过 SFTP 上传到 `/var/www/class-up/audio/`。
3. 上传完成后执行 `stat(remote_path)`，要求远端 `st_size` 等于本地 `size_bytes`。
4. 生成公网 URL：

```text
https://boneorbit.com/class-up/audio/{remote_filename}
```

5. 将 URL 写入豆包 submit 的 `audio.url`。
6. 将远端文件名、远端路径、公网 URL、上传时间写入 `manifest.segments[].remote_audio`。
7. 不把 SSH 私钥、服务器密码或 API Key 写入 manifest。

### 6.3 上传失败处理

上传失败时：

- 分段状态写入 `failed`。
- 错误码使用 `TRANSCRIPTION_FAILED`。
- `detail` 可以记录远端目录、文件名、本地大小、远端大小和异常类型，但不得记录密钥。
- `sftp.put()` 异常、连接异常、`stat` 失败或远端大小不一致，都必须转换为可重试 `UploadError`。
- 远端大小不一致时，必须尝试删除残缺远端文件；删除失败只记录 warning，不覆盖原始上传错误。
- 允许根据 `max_retries` 对“上传 + 豆包 submit/query + 结果转换”完整链路重试。
- 每次尝试都更新 `retry_count`，并在 `error.detail` 或 `reviews[]` 中记录 `attempt=x/y`、`error_type`、`retryable` 和可读 detail。

### 6.4 远端中转音频清理

远端音频只用于豆包服务端临时下载，不参与本地缓存复用。

- 豆包识别完成后，程序应调用 SFTP 删除本分段 `remote_audio.remote_path`。
- 删除成功时写入 `remote_audio.deleted_at`。
- 删除失败时写入 `remote_audio.delete_error`，并追加 manifest review 或 cleanup audit warning；不得影响分段转录成功状态。
- 后台清理仍按 `cleanup.remote_audio_ttl_hours` 扫描远端目录中过期项目音频，防止服务器空间被中转文件撑满。

## 7. 结果映射协议

豆包成功返回示例结构：

```json
{
  "audio_info": {
    "duration": 3696
  },
  "result": {
    "text": "这是字节跳动，今日头条母公司。",
    "utterances": [
      {
        "start_time": 0,
        "end_time": 1705,
        "text": "这是字节跳动，"
      }
    ]
  }
}
```

映射规则：

| 豆包字段 | 内部字段 | 规则 |
|---|---|---|
| `result.utterances[].start_time` | `items[].start` | 毫秒除以 1000，保留秒 |
| `result.utterances[].end_time` | `items[].end` | 毫秒除以 1000，保留秒 |
| `result.utterances[].text` | `items[].text` | 去除首尾空白，空文本丢弃 |
| 无稳定置信度字段 | `items[].confidence` | 写入 `null` |

内部转录结果：

```json
{
  "schema_version": "1.0",
  "segment_id": "segment-0001",
  "source_audio": "intermediate/segments/segment-0001.wav",
  "time_base": "segment_relative",
  "language": "zh",
  "provider": "doubao",
  "model": "bigmodel",
  "items": [
    {
      "item_id": "segment-0001-item-0001",
      "start": 0.0,
      "end": 1.705,
      "text": "这是字节跳动，",
      "confidence": null
    }
  ],
  "raw_output_path": "intermediate/transcription/raw/segment-0001.doubao.json",
  "error": null
}
```

兜底规则：

- 如果 `show_utterances` 未返回 `utterances`，但有 `result.text`，可以生成单条 item：
  - `start = 0.0`
  - `end = segment.end - segment.start`
  - `text = result.text`
- 兜底发生时必须在 manifest 的 `reviews[]` 中增加复核记录，类型为 `transcription_timestamp_fallback`。

## 8. 原始响应保存

每段豆包原始响应保存到：

```text
intermediate/transcription/raw/{segment_id}.doubao.json
```

保存内容：

- submit request id。
- submit 响应头中的 `X-Api-Status-Code`、`X-Api-Message`、`X-Tt-Logid`。
- query 最终响应头中的 `X-Api-Status-Code`、`X-Api-Message`、`X-Tt-Logid`。
- query 成功响应 body。

禁止保存：

- `X-Api-Key` 原文。
- SSH 私钥。
- 服务器登录密码。

## 10. 验收计划

### 10.1 协议级测试

- 豆包 `utterances` 转内部 `items[]`。
- 毫秒时间转换为秒。
- 空文本过滤。
- 缺失 `utterances` 时生成 review。
- 状态码映射为可重试或不可重试错误。
- SFTP 上传后远端大小校验。
- 上传失败、远端大小不一致和网络超时按分段完整重试。
- 远端中转音频删除成功/失败均可追踪，失败不影响主任务。
- manifest 和 raw 响应不包含 API Key。

### 10.2 联调测试

准备短测试音频：

```text
https://boneorbit.com/class-up/audio/test.wav
```

验证：

- submit 返回 `20000000`。
- query 最终返回 `20000000`。
- `result.utterances[]` 存在。
- 内部转录 JSON 的 `time_base` 为 `segment_relative`。
- `items[].start/end` 为秒，且 `end >= start`。

### 10.3 M1 端到端测试

使用 30 分钟以内测试视频：

- ffmpeg 提取标准化音频。
- 分段音频上传到云服务器。
- 上传后远端文件大小必须等于本地文件大小。
- 豆包逐段转录。
- 转录完成后远端中转音频应被主动删除，删除失败只记录警告。
- 合并生成 `{原文件名}_Subtitles[数字].srt`。
- 合并生成 `{原文件名}_text[数字].txt`。
- manifest 中 `stages.m1.status == "success"`。
- manifest 和日志不含 API Key 原文。
