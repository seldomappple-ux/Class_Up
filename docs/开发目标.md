# 网课助手开发目标

## 1. 产品目标

本项目用于将网课录屏加工为可检索、可复习、可切片复用的学习资料包。

系统输入一段网课录屏后，自动完成音频提取、转录、字幕生成、全文文稿生成、知识点结构化、视频切片和素材目录整理，最终输出一套可直接归档、导入知识库或继续编辑的视频学习资料。

## 2. 输入与输出

### 2.1 典型输入

- 网课录屏文件，单个文件时长原则上不超过 2 小时
- 模型服务配置，包括 API Key、服务地址、模型名称、并发限制和上传限制
- 可选的课程元信息，例如课程名、讲师、章节名、标签

### 2.2 典型输出

- 完整字幕文件：`full_subtitle.srt`
- 完整文本文稿：`full_transcript.txt`
- 知识库版文稿：`knowledge_base.md`
- 知识点结构化数据：`knowledge_points.json`
- 知识点切片视频
- 每个知识点对应的学习包目录
- 任务执行记录：`manifest.json`

## 3. 首版范围

首版必须完成：

- 本地视频文件处理
- M1 音频准备能力，包括一键视频转音频、音频提取和分段
- 转录模型服务适配
- SRT 字幕生成
- 完整文稿生成
- 文本分析模型服务适配
- 知识点结构化
- 按知识点切割视频
- 本地学习包目录整理

首版不处理：

- 在线课程平台账号登录、自动下载或 DRM 绕过
- 多人说话人精确分离
- 自动生成练习题、测验题或思维导图
- 多用户权限、团队协作和云端任务队列
- 对外 SaaS 化部署

## 4. 总体处理流程

```text
输入视频
  -> ffmpeg 提取音频
  -> 音频标准化与分段
  -> 调用转录模型服务
  -> 合并分段转录结果
  -> 输出 SRT 字幕和完整文稿
  -> 调用文本分析模型服务
  -> 生成知识点结构化数据和知识库文稿
  -> ffmpeg 按知识点时间范围切割视频
  -> 组织课程学习包目录
  -> 写入 manifest 执行记录
```

## 5. 架构原则

- 业务流程不绑定具体模型供应商。
- 转录模型和文本分析模型通过适配层接入。
- 外部模型返回值必须先转换成项目内部统一结构。
- ffmpeg 调用统一封装，避免业务代码直接拼接复杂命令。
- 每个阶段都要有可落盘的中间产物，便于失败后恢复和排查。
- 输出目录结构必须稳定，避免后续知识库导入和人工整理成本上升。

## 6. 技术约束

| 层 | 选型 |
|---|---|
| 语言 | Python 3.11+ |
| 音视频处理 | ffmpeg |
| ffmpeg 调用 | Python `subprocess` 封装 ffmpeg CLI |
| 转录 | 可配置的专用转录模型服务，优先支持国产模型 |
| 文本分析 | 可配置的国产大模型 / 专用文本分析模型 |
| 配置 | `.env` + YAML / JSON 配置文件 |
| 数据结构 | `pydantic` |
| CLI | `argparse`，后续可升级为 `typer` |
| 测试 | `pytest` |
| UI | Gradio，可选 |

## 7. 模型接入要求

项目不固定具体模型名称。模型名称、服务地址、鉴权方式、上传限制、并发限制、超时时间和重试策略全部通过配置提供。

模型适配层至少支持两类能力：

- 转录模型：输入音频分段，输出带时间戳的文本片段。
- 文本分析模型：输入完整文稿和字幕时间轴，输出知识点结构化数据。

## 8. 建议模块边界

```text
src/
  class_up/
    cli.py
    config.py
    manifest.py
    media/
      ffmpeg.py
      audio.py
      video.py
    transcription/
      base.py
      service.py
      merge.py
      srt.py
    analysis/
      base.py
      service.py
      knowledge_points.py
    packaging/
      organizer.py
      naming.py
    utils/
      timecode.py
      filesystem.py
```

| 模块 | 职责 |
|---|---|
| `cli.py` | 命令行入口，接收输入视频、输出目录和配置文件 |
| `config.py` | 加载 `.env` 与配置文件，生成运行配置 |
| `manifest.py` | 记录任务输入、阶段状态、中间产物、错误信息 |
| `media/ffmpeg.py` | 统一封装 ffmpeg 命令执行 |
| `media/audio.py` | 视频转音频、音频提取、标准化、切段 |
| `media/video.py` | 视频切片、字幕挂载 |
| `transcription/base.py` | 转录模型适配接口 |
| `transcription/service.py` | 调用具体转录服务 |
| `transcription/merge.py` | 合并分段转录结果，处理重叠区 |
| `transcription/srt.py` | 生成 SRT 字幕 |
| `analysis/base.py` | 文本分析模型适配接口 |
| `analysis/service.py` | 调用具体文本分析服务 |
| `analysis/knowledge_points.py` | 校验和规范化知识点结构 |
| `packaging/organizer.py` | 生成最终课程学习包 |
| `packaging/naming.py` | 文件和目录命名规范 |
| `utils/timecode.py` | 时间码转换 |
| `utils/filesystem.py` | 路径、文件名清理和目录创建 |

## 9. 核心数据契约

### 9.1 转录片段

```json
{
  "segment_id": "segment-0001",
  "start": 0.0,
  "end": 12.5,
  "text": "本节课我们介绍梯度下降的基本思想。",
  "source_audio": "segments/segment-0001.wav"
}
```

### 9.2 知识点

```json
{
  "index": 1,
  "title": "梯度下降原理",
  "start": 750.0,
  "end": 1125.0,
  "keywords": ["梯度", "学习率", "损失函数"],
  "summary": "本段介绍梯度下降如何通过迭代更新参数降低损失。",
  "content": "..."
}
```

### 9.3 Manifest

`manifest.json` 至少记录：

- 输入视频路径和基础元信息
- 输出目录
- 运行配置摘要
- 分段列表
- 每个阶段的状态：`pending / running / success / failed`
- 中间产物路径
- 错误信息和重试次数
- 任务开始时间和结束时间

## 10. 输出目录规范

```text
课程名/
  manifest.json
  full_transcript.txt
  full_subtitle.srt
  knowledge_base.md
  knowledge_points.json
  segments/
    segment-0001.wav
    segment-0002.wav
  clips/
    01_梯度下降原理_00-12-30/
      clip.mp4
      subtitle.srt
      notes.md
    02_反向传播_00-18-45/
      clip.mp4
      subtitle.srt
      notes.md
```

命名要求：

- 目录和文件名需要清理非法字符。
- 知识点目录必须带序号，避免排序混乱。
- 时间码统一使用 `HH-MM-SS`。
- 中间产物和最终产物分层保存，避免覆盖原始输入。

## 11. 安全要求

- API Key 不写入代码和文档样例。
- 本地密钥放入 `.env`，并确保后续加入 `.gitignore`。
- 配置文件只保存非敏感默认值。
- 日志和 manifest 不能完整记录密钥。
- 模型服务超时、重试次数和并发数必须可配置。

## 12. CLI 命令约定

当前后端至少提供以下命令：

```powershell
python -m class_up.cli audio 输入视频.mp4 --output 输出音频.wav --overwrite
python -m class_up.cli m1 输入视频.mp4 --config config\config.example.yaml --output-root outputs --course-title 课程名
```

约定：

- `audio` 归属于 M1 音频准备能力，用于单独执行 M1 的视频转音频子步骤。
- `audio` 默认输出 `wav`、`16000 Hz`、单声道音频。
- `audio --output` 指定完整输出文件路径；`audio --output-dir` 指定输出目录并使用视频同名音频文件。
- `audio` 只执行 M1 音频准备中的转换动作，不触发分段、转录、合并。
- 完整 M1 流水线仍通过 `m1` 命令创建标准输出目录并写入 `manifest.json`。
- `m1` 是完整 M1 转录流水线入口；旧式 `python -m class_up.cli 输入视频.mp4 ...` 暂时保留兼容。
