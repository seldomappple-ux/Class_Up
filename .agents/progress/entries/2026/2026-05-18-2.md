Established the initial folder structure for a pure software project.

The base structure includes `src/`, `tests/`, `docs/`, `config/`, `scripts/`, `examples/`, and `tools/`, with focused subdirectories for unit and integration tests plus requirements, design, and API documentation.

This structure intentionally avoids hardware-specific folders, comparison-test phase folders, and multi-IDE extension layers.
