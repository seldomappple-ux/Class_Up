## Summary

Two M1 output improvements were implemented after real playback feedback:

- Subtitle display was too dense because one SRT block could contain multiple spoken sentences.
- Output filenames were still internal pipeline names, and repeated transcription of the same source video wasted cutting/upload/transcription work.

M1 output files now use the original uploaded video filename as the base name instead of fixed `full_subtitle.srt` and `full_transcript.txt`.

Example:

```text
lesson_Subtitles.srt
lesson_text.txt
lesson_Subtitles_2.srt
lesson_text_2.txt
```

The run suffix is used when the same course/output directory is generated again.

## Subtitle Readability Change

- SRT rendering now splits long multi-sentence subtitle entries into shorter subtitle blocks.
- Splitting prefers Chinese/English punctuation boundaries and caps each block around normal subtitle length.
- Text transcript output remains unchanged; only `full_subtitle` rendering is split for readability.
- Added tests in `tests/unit/test_srt.py` for long multi-sentence subtitle splitting and monotonic timing.

## Output Naming and Cache Change

- Manifest now records `input.source_stem` and `media.source_video.sha256`.
- Output directories avoid overwriting existing runs by using `_2`, `_3`, etc.
- `write_m1_outputs()` writes source-based subtitle and transcript names.
- Web download buttons now use the real generated filenames from manifest outputs.
- Web uploads pass the original browser filename into manifest creation.
- M1 Web pipeline can reuse completed intermediate audio, segments, and transcription JSON when source video sha256 and relevant config match.
- Added tests in `tests/unit/test_transcription_merge.py` and `tests/unit/test_m1_cache.py`.

## Durable Lesson

Generated subtitle and transcript filenames should follow the user's source media, not internal pipeline names. Subtitles need their own readability pass because ASR utterance boundaries are not always suitable display boundaries. Re-running the same source should not spend upload/transcription traffic again when the source hash and transcription config are identical.
