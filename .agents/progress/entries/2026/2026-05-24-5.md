## Summary

M1 subtitle timing now handles two real validation failures:

- Extracted ASR WAV could be shorter than the source MP4 audio timeline.
- Reading-friendly SRT splitting estimated child timings and could make sentence-level review confusing.

The fix keeps the existing simple subtitle pipeline and does not add AI alignment.

## Changes

- `ffmpeg.extract_audio()` now uses `aresample=async=1:first_pts=0` while keeping WAV PCM 16 kHz mono output.
- M1 audio preparation records source media duration, source audio stream duration, normalized WAV duration, duration delta, sync status, and correction ratio in `manifest.media.normalized_audio`.
- Duration drift above `0.2s` writes an `audio_timeline_drift_warning` review instead of failing the task.
- Legacy resumed or cache-reused tasks fill missing timeline diagnostics before merge.
- Transcription merge applies linear timeline correction only to final merged subtitle times when drift is detected; raw provider JSON is unchanged.
- SRT readability splitting is locked inside the parent ASR item: first child starts at parent start, last child ends at parent end, short parents are not force-split, and split metadata is kept in memory for tests/debugging.

## Lesson

Frame-accurate subtitle timing requires validating media timeline preservation, not just SRT monotonicity. Reading-friendly subtitle splitting must be treated as an internal display pass bounded by the ASR parent item.
