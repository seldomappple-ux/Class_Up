Added an M1 audio preparation CLI command for one-click video to audio conversion.

The new command is `class-up audio <video>`. It belongs to the M1 audio preparation capability. It defaults to 16kHz mono wav, supports `--output` for an exact output file path, supports `--output-dir` for directory-based output, and requires `--overwrite` when the destination file already exists.

The previous M1 invocation style remains compatible. The explicit subcommand `class-up m1 <video>` is also available for the full M1 transcription pipeline.

Verification completed:

- `python -m pytest`
- `$env:PYTHONPATH='src'; python -m class_up.cli audio --help`
- `$env:PYTHONPATH='src'; python -m class_up.cli audio outputs\test_inputs\m1_backend_smoke.mp4 --output outputs\audio_smoke\m1_backend_smoke.wav --overwrite`
- `ffprobe -v error -show_entries stream=codec_type,sample_rate,channels -of json outputs\audio_smoke\m1_backend_smoke.wav`

The smoke output was verified as an audio stream with sample rate `16000` and `1` channel.
