## Summary

Added a layered cleanup system so temporary transport files, local uploaded videos, reusable intermediate caches, and final outputs are handled differently.

Defaults:

- Remote SFTP temp audio: 6 hours
- Successful local uploaded video: 1 day
- Failed local uploaded video: 7 days
- Local `intermediate/` cache: 7 days
- Final outputs and `manifest.json`: protected
- Disk pressure threshold: 10GB free space

## Changes

- Added `CleanupConfig` and `cleanup:` section in `config/config.example.yaml`.
- Added `src/class_up/cleanup.py` for cleanup planning, dry-run, execution, and audit logging.
- Added `outputs/system/cleanup_events.jsonl` audit output.
- Added CLI commands: `class-up cleanup --dry-run` and `class-up cleanup --execute`.
- Added Web endpoints: `GET /cleanup/preview` and `POST /cleanup/run`.
- Added SFTP `delete_audio()` and remote upload metadata on transcription segments.
- Doubao segment processing now attempts remote temp audio cleanup after upload usage; failures are recorded as warnings and do not fail the task.
- Cache reuse now requires intermediate audio and transcription JSON files to still exist.

## Durable Lesson

Cleanup should follow artifact value, not a single global age. Transport files can disappear quickly, uploaded source videos are short-lived diagnostics, intermediate caches preserve retry and reuse value for a limited window, and final outputs should stay unless the user explicitly chooses otherwise.
