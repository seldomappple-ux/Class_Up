Defined the M1 Doubao ASR integration protocol using the new console API Key flow.

The project will use `X-Api-Key`, resource id `volc.seedasr.auc`, standard submit/query endpoints, SFTP upload to `/var/www/class-up/audio/`, and public URLs under `https://boneorbit.com/class-up/audio/`.

The protocol records how Doubao `result.utterances[]` maps to internal `items[]`, how milliseconds convert to segment-relative seconds, how status codes map to internal errors, and how raw responses are stored without leaking secrets.

Config and code were aligned so `config/config.example.yaml` can load the new `transcription.resource_id`, polling fields, request options, and `upload` section.
