## Summary

M1 backend now has a real `doubao` transcription provider path behind the existing provider selector. Selecting Doubao from the Web UI uses the entered API Key as `CLASS_UP_DOUBAO_API_KEY`, defaults endpoint/model/resource ID to the新版控制台录音识别配置, enables SFTP upload, then submits and polls Doubao ASR.

The Web UI provider dropdown now includes `豆包语音识别`, validates that an API Key is provided for Doubao, and keeps mock mode available for local smoke tests.

## Files Changed

- `src/class_up/transcription/doubao.py` — Doubao submit/query adapter, raw response persistence, `utterances[]` to internal `items[]` conversion.
- `src/class_up/transcription/service.py` — provider dispatch for `doubao`, fallback review handling, retryable Doubao error mapping.
- `src/class_up/upload/` — upload service interface and SFTP implementation.
- `src/class_up/api/jobs.py` — Web job config defaults for Doubao, including `CLASS_UP_DOUBAO_API_KEY` and SFTP upload provider.
- `src/class_up/api/app.py` — server-side API Key validation for Doubao jobs.
- `static/index.html` — repaired Web UI text and added Doubao provider selection flow.
- `pyproject.toml` — added `httpx` and `paramiko`.
- `tests/unit/test_doubao_transcription.py` and `tests/unit/test_web_config.py` — provider mapping and config coverage.
- `docs/M1开发阶段/豆包录音识别接入协议.md` — synced Web UI Doubao selection, current implementation files, and remaining live validation.
- `docs/M1开发阶段/M1阶段开发计划.md`, `docs/开发计划.md`, `docs/数据协议.md`, `config/README_中文.md`, `README.md` — aligned docs with the implemented provider path.
