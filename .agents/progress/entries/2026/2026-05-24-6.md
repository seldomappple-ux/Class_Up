## Summary

Doubao `show_utterances=true` can return `utterances[].words[]` with word-level timing. M1 now preserves those word timings and uses them to align split subtitle chunks when possible.

This fixes cases where one utterance contains a long pause: the second subtitle chunk now starts from the matching word timestamp instead of an estimated text-ratio position.

## Changes

- Internal ASR items preserve optional `words[]` with `text`, `start`, `end`, and `blank_duration` in seconds.
- Merge carries word timestamps onto the corrected original-video time axis.
- SRT splitting matches chunks to words sequentially with normalized text matching.
- If word matching fails for a parent utterance, the whole parent falls back to the previous proportional timing strategy.
- SRT rendering removes commas, periods, dunhao, and semicolons by default while keeping question and exclamation marks.
- Transcript and future knowledge-base text still use original punctuation.

## Lesson

Subtitle display text and transcript text are separate products. SRT can be punctuation-light and timing-sensitive, while transcript and knowledge-base content should preserve readable punctuation.
