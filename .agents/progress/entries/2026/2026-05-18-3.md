Revised the course assistant development plan to avoid binding the workflow to fixed OpenAI or Claude model names.

The plan now treats transcription and text analysis as configurable model services, with domestic dedicated models as the intended direction. A Phase 0 was added for ffmpeg checks, configuration, model adapters, manifest generation, and minimal pipeline validation.

This keeps the business pipeline stable while allowing the actual model provider, model name, endpoint, authentication, and rate limits to be selected later through configuration.
